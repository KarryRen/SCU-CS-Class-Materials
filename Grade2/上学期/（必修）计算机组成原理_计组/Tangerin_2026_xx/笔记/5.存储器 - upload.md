# 2.Cache
通常由**SRAM（静态）** 构成
## 2.1.直接映射
$$
块地址=Tag×Cache块数+Index
$$

$$
字节地址=Tag×(Cache块数)×(块大小)+Index×(块大小)+Offset
$$

主存里的每个块，只能放到 Cache 里唯一固定的一行。
主存中每个数据块，通过一种映射方式，只能对应cache中的一个位置：
```
Index=块地址 mod cache中的块数
Tag=块地址/cache中的块数 并向下取整
```

例子：主存块地址 = 13，Cache 有 4 块
那么13 mod 4 = 1，表示这个主存块只能放到 Cache 的第 1 块（从 0 开始编号时）。
13/4取整=3，表示这个主存块的tag是3
多个主存数据块对应一个cache位置

#### 它怎么工作
一个内存地址通常会被分成 3 部分：
1. **标记（Tag）**：用来判断是不是你要的数据
2. **索引（Index）**：决定它只能去 Cache 的哪一行
3. **块内偏移（Offset）**：表示这个块里的具体位置。如果每块只有一个字，那就不需要offset
#### 访问过程
当 CPU 想读一个数据时：例子见[p.11](大二春季学期/计算机组成原理/第五章教材.pdf#page=11&selection=22,0,24,2)
1. 先根据 **Index** 找到 Cache 的那一行
2. 如果有效位V是Y(Yes)，进入下一步；否则直接第五步miss
3. 看这行里的 **Tag** 是否匹配
4. 匹配：**命中（hit）**，直接读
5. 不匹配：**失效（miss）**，去主存找，然后从主存把数据搬进来，覆盖原来那一行（因为要找的数据使用频率可能比较高）, 然后把有效位V置为有效，更新tag。
#### 位数
例题见：[p.13](大二春季学期/计算机组成原理/第五章教材.pdf#page=13&selection=139,0,142,4)
**cache位数计算公式**   <font color=red>(这个公式容易遗漏)</font>：
$$
块数×(块大小+标记字段Tag位数+有效位V字段位数1)
$$
**标记字段计算方法**：
1. 先算块内偏移 Offset
    - 块大小 = $2^m$ 字节
    - 那么 Offset = m 位
2. 再算 Cache 行号 Index
    - Cache 一共有 $2^n$ 块
    - 那么 Index = n 位
3. 最后算 Tag
    - Tag = 地址总位数 − Index − Offset
    - <font color=orange>地址总位数：用总内存大小来算，比如1MB就对应20位，1GB对应30位</font>

其他题型：[p.13](大二春季学期/计算机组成原理/第五章教材.pdf#page=13&selection=263,0,267,2)
PPT上还有大量例题（但是PPT不一定发出来了）

## 2.2.写操作处理
写操作有两种基本策略：
#### 写直达
[将这个数据同时写入主存和 cache 中](大二春季学期/计算机组成原理/第五章教材.pdf#page=16&selection=78,0,82,1)
- 好处：Cache 和主存更容易保持一致
- 坏处：很慢，因为每次写都要等主存
为什么会存在不一致：**Cache 追求速度，主存追求容量；为了快，数据更新常常不是同步完成，所以会出现短暂不一致**

#### 写回
[新值仅仅被写入 cache 块中](大二春季学期/计算机组成原理/第五章教材.pdf#page=16&selection=209,12,213,2&color=significant)，主存先不改，等这个 Cache 块以后被替换掉时，才把修改后的内容写回主存，可以理解成：先改草稿，等最后再统一誊写正式稿。
- 好处：**快**
- 坏处：实现更复杂，需要记录这个块有没有被改过

**写缺失**：CPU 想写某个数据，但这个数据所在的块不在 Cache 里。[p.16](大二春季学期/计算机组成原理/第五章教材.pdf#page=16&selection=98,17,98,20&color=significant)
常见处理方式是：
1. 先从主存把整个块取到 Cache
2. 再把要写的那个字写进去
3. 有时也会顺便更新主存

**写缓冲**：[p.16](大二春季学期/计算机组成原理/第五章教材.pdf#page=16&selection=167,14,167,17&color=significant)


## 2.3.cache的其他映射方式
#### 全相联映射
和直接映射一样，和都是在说：  主存中的一个块，到了 Cache 里该放哪儿。
[一个块可以被放置在 cache 中的任何位置](大二春季学期/计算机组成原理/第五章教材.pdf#page=22&selection=75,9,79,6&color=significant)
要在全相联 cache 中查找一个指定的块，由于该块可能被存放在 cache 中的任何位置，因此需要检索 cache 中所有的项。为了使检索更加有效，检索由一个与 cache 中每个项都相关的比较器并行完成。这些比较器加大了硬件开销，因而，全相联只适合块数较少的 cache

#### 组相联映射
介于全相联和直接映射之间
**一个主存块只能放到某一“组”里，但在这组内部可以放到任意一行。**
举例：如果是 **4 路组相联**：
- Cache 被分成很多组
- 每组有 4 个位置
- 一个主存块只能去指定的那一组
- 但在这一组的 4 个位置里，随便放一个
**计算公式：**
$$
组号=块号 mod   组数
$$

## 2.5.虚拟存储器
[p.40](大二春季学期/计算机组成原理/第五章教材.pdf#page=40&selection=34,0,34,5)
操作系统给每个程序“画”出来的一大片连续内存空间。程序以为自己拥有很多、连续的内存，但实际上这些数据可能：一部分在内存 RAM 里, 一部分在硬盘/SSD 里
不够用时把暂时不用的数据放到磁盘上
#### 访问
虚拟存储器采用分页存储
程序看到的是虚拟存储器的虚拟地址，通过页表转换为物理地址访问存储
如果要访问的页不在RAM中,会出现缺页异常。此时需要页面置换
要加快地址转换，使用[快表](../操作系统/4.存储器管理.md#^a4a279)（详细参考操作系统的内容）
**虚拟存储系统一般采用全相联**，减少缺页，增加利用率
#### 存在位/有效位
 - 为1：该页已经在RAM中，可以直接访问，不会缺页
 - 为0：反之

## 2.7.主存储器校验
避免bit翻转等情况
**码距**（汉明距离）：同一编码方式，任何两组二进制代码中，对应位置的代码最少有几个二进制位不同。比如1100和1110的码距就是1，1111和0000的码距是4
	码距与发现/纠正错误位数的关系：^777717
		检错：发现 **e** 个错误，码距≥ e + 1
		纠错：纠正 **t** 个错误，码距 ≥ 2t + 1 ^65ff08
#### 奇偶校验
加一个二进制位，使得1的个数是奇数/偶数（奇校验/偶校验）.码距为2（码距要求同一编码方式，单一奇校验/偶校验只改1位肯定不合法）
#### 汉明校验
[p.34](大二春季学期/计算机组成原理/第五章教材.pdf#page=34&selection=73,0,74,1)
相比奇偶校验，能做到：发现错误，并指出错在哪一位，然后改回来
最小码距是3（码距是1无法检测，码距是2（比如奇偶校验）只能检测不能纠错）
1. 校验位放在哪里：放在编号为 2 的幂的位置
2. 校验位怎么分组：看位置编号的二进制，（以七位二进制为例，其余同理）
	P1 检查编号二进制最低位为 1 的位置（P1 检查：1, 3, 5, 7，因为1=001，3=011，5=101，7=111，最低位都是1）
	P2 检查编号二进制第 2 位为 1 的位置（P2 检查：2, 3, 6, 7）
	P4 检查编号二进制第 3 位为 1 的位置（P4 检查：4, 5, 6, 7）‘
	每一组内做奇偶校验（默认是偶校验）
	每一个校验位校验的数据位数不一定一样
3. 出错时怎么定位：接收端重新做校验，把每一组的几个数字做异或分别得到S1, S2, S4，把它们组成一个二进制数
	`S4 S2 S1`,这个数就是出错的位置。
	例如：S4 S2 S1 = 101=十进制 5。说明：第 5 位出错, 把第 5 位取反即可纠正。
	如果没错，S4 S2 S1就是全0
4. 汉明校验位需要多少位
	如果有 `k`位数据，需要` r `位校验位，需要满足：$2^r >= k + r + 1$
	因为 r 位校验结果要能表示：无错误 + 每一位出错的位置
	总共 k + r 位，再加 1 种“无错”情况。

# 3.一些补充
## 3.1.SRAM & DRAM
### 3.1.1.SRAM — 静态

**实现原理**：SRAM 存储单元由**多个晶体管（通常 6 个）组成触发器电路**来存储 1 位数据。

```
         Vdd
          |
        Load P1    Load P2
          |          |
    bit —|— M3    M4 —|— bit反
          |          |
        M1 ———— M2
          |          |
         GND        GND
```

- 使用**双稳态触发器**（cross-coupled inverters）锁存数据
- 只要不断电，数据就不会丢失（不需要刷新）
- 每个存储单元通常需要 6 个晶体管（6T SRAM）

**特点**：
- 速度快（访问时间短，通常 1-10 ns）
- 不需要刷新操作
- 功耗相对较高（静态功耗）
- 集成度低（每个单元晶体管多，占用面积大）
- 成本高
- **通常用于 Cache**

### 3.1.2.DRAM — 动态

**实现原理**：DRAM 使用**一个晶体管 + 一个电容**（1T1C）来存储 1 位数据。

```
        字线（Word Line）
            |
          |—|
          | T |
          |—|
            |
            |
           ===  电容（存储电荷）
            |
           GND
```

- 电容充电（有电荷）= 1，电容放电（无电荷）= 0
- **电容会漏电**，电荷会逐渐流失，所以需要**周期性刷新**（通常每 64ms 刷新一次）
- 每个存储单元只需 1 个晶体管 + 1 个电容

**特点**：
- 密度高（单元小，同样面积可集成更多位）
- 成本低
- 功耗相对较低（动态功耗）
- 需要**刷新**（refresh）电路
- 速度较慢（访问时间约 50-100 ns）
- **通常用于主存**

### 3.1.3.SRAM vs DRAM 对比总结

| 对比项 | SRAM | DRAM |
|--------|------|------|
| **存储原理** | 触发器锁存（双稳态电路） | 电容存储电荷 |
| **单元结构** | 6 个晶体管 | 1 个晶体管 + 1 个电容 |
| **是否需要刷新** | 否 | 是（约 64ms 周期） |
| **访问速度** | 快（~1-10 ns） | 较慢（~50-100 ns） |
| **集成度** | 低 | 高 |
| **功耗** | 较高（静态功耗大） | 较低（动态功耗，但刷新需要额外功耗） |
| **成本** | 高 | 低 |
| **用途** | Cache（L1/L2/L3） | 主存 |

**核心区别一句话**：SRAM 用触发器锁存数据，速度快但贵；DRAM 用电容存储数据，密度高便宜但需要刷新。

> 补充：刷新操作时，DRAM 控制器会先读出每个单元的数据，再重新写回去，以恢复电容中被放掉的电荷。

## 3.2.存储器的局部性原理

**局部性原理（Principle of Locality）**：程序在执行过程中，对指令和数据的访问并不是均匀随机的，而是呈现出**聚集于某个局部区域**的倾向。

### 3.2.1.时间局部性

**定义**：如果一个存储位置被访问过，那么在**不久的未来很可能再次被访问**。

### 3.2.2.空间局部性

**定义**：如果一个存储位置被访问过，那么**其附近的存储位置很可能在不久后也被访问**。

**典型例子**：
- **数组遍历**：数组元素在内存中是连续存放的
  ```c
  // 按行遍历，利用了空间局部性
  for (int i = 0; i < 1000; i++) {
      sum += a[i];  // a[i] 和 a[i+1] 在内存中相邻
  }
  ```
- **顺序执行的指令**：程序指令通常顺序存放，CPU 会顺序取指执行
- **结构体/对象字段访问**：同一对象的字段在内存中相邻存放

**利用方式**：
- Cache 每次从主存读取数据时，不是只读一个字，而是**读入一个块（block/cache line）**，把目标数据及其附近的数据一起搬进 Cache
- 预取（prefetch）机制：预测接下来可能用到的数据提前加载

### 3.2.3.两种局部性的关系

| | 时间局部性 | 空间局部性 |
|--|-----------|-----------|
| **关注点** | 同一地址的重复访问 | 相邻地址的访问 |
| **利用手段** | 把数据留在 Cache 中不换出 | 一次读入整个块 |
| **典型场景** | 循环、频繁访问的变量 | 数组顺序遍历、顺序执行指令 |

### 3.2.4.为什么局部性原理很重要

局部性原理是**存储层次结构能够有效工作的理论基础**：

1. **Cache 的设计**：Cache 利用时间局部性（保留已用数据）和空间局部性（预取相邻数据）来提高命中率
2. **虚拟内存**：利用局部性原理，程序运行时可只将当前需要的一小部分页面加载到物理内存中
3. **TLB（快表）**：也依赖局部性来缓存地址转换结果
