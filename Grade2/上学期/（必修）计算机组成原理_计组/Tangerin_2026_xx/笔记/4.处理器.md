# X1.单周期控制信号

## `RegDst`

决定写入哪个寄存器

|取值|含义|什么时候|
|---|---|---|
|0|写 `rt`，即 `Instruction[20-16]`|`lw`|
|1|写 `rd`，即 `Instruction[15-11]`|R-type，包括 `slt`|
|X|不写寄存器，所以无所谓|`sw`, `beq`, `j`|

| 1   | 是条件分支   | `beq`                   |
| X   | 一般不用写 X | 无                       |

# X.2.多周期控制信号
在多周期实现中，控制信号不仅取决于指令opcode，还取决于**当前时钟周期**（即FSM状态）。以下列出多周期中新增的控制信号。
## `IorD`
决定存储器地址来自Instruction（PC）还是Data（ALU）

|取值|含义|什么时候|
|---|---|---|
|0|地址来自PC（取指令）|所有指令的第1周期（IF）|
|1|地址来自ALUOut（访数据）|`lw`/`sw`的第4周期（MEM）|
|X|不访问存储器时无所谓|其他周期|

## `ALUSrc`
| 信号          | 位数      | 解释                             |
| ----------- | ------- | ------------------------------ |
| **ALUSrcA** | **1 位** | 控制 ALU 的第一个输入（A 端）的 MUX        |
| **ALUSrcB** | **2 位** | 控制 ALU 的第二个输入（B 端）的 MUX（4 选 1） |
### ALUSrcA

| 取值    | 送到 ALU 的 SrcA             | 用在哪个阶段/场景                   |
| ----- | ------------------------- | --------------------------- |
| **0** | **PC**（当前程序计数器的值）         | IF阶段计算 PC+4；ID分支阶段计算跳转地址    |
| **1** | **A 寄存器**（寄存器堆 RD1 的输出暂存） | EX阶段R的A op B；计算 lw/sw 的内存地址 |
### ALUSrcB
| 取值     | 送到 ALU 的 SrcB                   | 用在哪个阶段/场景                   |
| ------ | ------------------------------- | --------------------------- |
| **00** | **B 寄存器**（寄存器堆 RD2 的输出暂存）       | A op B，R-type的EX阶段          |
| **01** | **常数 4**                        | IF阶段计算 `PC+4`               |
| **10** | **符号扩展的立即数** SignExt(imm)       | lw/sw 计算基地址+偏移；addi 等 I 型运算 |
| **11** | **符号扩展后左移 2 位** SignExt(imm)<<2 | beq计算分支目标地址                 |

| 阶段 / 用途         | ALUSrcA | ALUSrcB | ALU 做什么        |
| --------------- | ------- | ------- | -------------- |
| 取指 IF           | 0       | 01      | `PC + 4`       |
| 译码 ID / 分支地址计算  | 0       | 11      | `PC + 偏移量`     |
| R 型运算           | 1       | 00      | `寄存器A 运算 寄存器B` |
| Load/Store 地址计算 | 1       | 10      | `基址寄存器 + 偏移量`  |
| 条件分支比较          | 1       | 00      | 比较两个寄存器        |

## `PCSource`
决定写入PC的值来源（**2位**信号）

| 取值  | 含义                              | 什么时候       |
| --- | ------------------------------- | ---------- |
| 00  | ALU结果（即PC+4）                    | 第1周期（PC+4） |
| 01  | ALUOut（分支目标地址）                  | `beq`跳转    |
| 10  | 跳转地址 `{PC[31-28], IR[25-0]<<2}` | `j`算地址     |
| XX  | 不写PC时无所谓                        | 其他周期       |

## `IRWrite`
写指令寄存器。

|取值|含义|什么时候|
|---|---|---|
|0|不写IR|除IF外的所有周期|
|1|写IR ← Memory[PC]|第1周期（IF）|

## `PCWrite`
无条件写PC。

| 取值  | 含义         | 什么时候                   |
| --- | ---------- | ---------------------- |
| 0   | 不写PC       | —                      |
| 1   | 写PC        | 第1周期（所有指令PC+4），`j`第3周期 |
| 0   | 写PC由Cond控制 | `beq`第三周期              |
## `PCWriteCond`
有条件写PC，和`beq`配合使用。

| 取值  | 含义           | 什么时候         |
| --- | ------------ | ------------ |
| 0   | 不根据条件写PC     | 除`beq`外的所有周期 |
| 1   | 若 A = B 则写PC | `beq`第三周期    |

## 继承自单周期的信号
以下信号在多周期中也存在，但取值随周期变化（而不仅是随指令）：

|信号|多周期变化说明|

# X.3.多周期指令
[p.66](大二春季学期/计算机组成原理/PPT/C4_Processor.pdf#page=66&selection=6,0,6,33&color=yellow)
执行时间较少的指令可以分配较少的时钟周期数
需要：
- **在每个功能单元后增加寄存器**
- **把指令执行拆成多个周期**
- **控制信号的取值**
- 把这些内容整合起来
## 3.1.多周期数据通路
![p.77](大二春季学期/计算机组成原理/PPT/C4_Processor.pdf#page=77&rect=15,184,708,447&color=yellow)
**和单周期的区别**：

### 新增的寄存器
由于指令执行被拆成多个周期，需要在各功能单元之后插入**暂存器**来保存中间结果，供下一个周期使用：^888fff ^cc47c1

| 寄存器 | 全称 | 位宽 | 作用 |
|---|---|---|---|
| **IR** | Instruction Register | 32位 | 保存当前正在执行的指令（第1周期从内存读出后锁存） |
| **MDR** | Memory Data Register | 32位 | 保存从数据内存读出的数据（lw指令第4周期），第5周期写回寄存器 |
| **A** | — | 32位 | 保存从寄存器堆读出的 `rs` 值（即 `Reg[IR[25-21]]`） |
| **B** | — | 32位 | 保存从寄存器堆读出的 `rt` 值（即 `Reg[IR[20-16]]`） |
| **ALUOut** | — | 32位 | 保存 ALU 计算结果（地址/运算结果/分支目标地址），供后续周期使用 |

> 这些寄存器是**单周期中没有的**。单周期的所有操作在一个时钟周期内走完，不需要暂存中间结果。

### 合并/移除的硬件
- **只有一个ALU**，没有单周期中的两个独立加法器（PC+4 和分支地址计算都复用这个ALU）
- **指令存储器和数据存储器合并**为一个统一存储器，通过 `IorD` 信号区分访问来源
- 因此新增了多个**多路复用器（MUX）**来在不同周期间切换数据来源
 
 **设计思路**：
**1. 重用功能单元（Reusing functional units）**
- 把指令的执行过程拆分为多个更小的步骤，在多个时钟周期中逐步完成指令的执行。
- 为了让所有步骤都能使用相同的功能单元（比如 ALU 或存储器），需要引入更多的多路复用器（MUX），以便不同步骤重用硬件资源。
**2. 使用寄存器保存中间结果 (Keep results in registers)**
- 每完成一个时钟周期后，会将计算结果暂时保存在流水线寄存器里。
- 在下一周期中，可以继续用这些寄存的中间结果来完成后续的计算。
**3. 控制信号的不再只取决于指令位**
- 单周期设计中，控制信号一般只从指令中直接解码得到。
- 但在多周期设计中，控制信号还要**根据当前的周期所处状态**来决定（因为每个周期做的事情完全不同）。
**4. 控制信号由有限状态机 (FSM) 生成**
- 这里会引入一个有限状态机（FSM），根据当前的系统状态和输入条件生成正确的控制信号，指导数据通路里的各个部件（MUX、ALU等）如何运行。

## 各步骤

#### 1: IF — 取指（Instruction fetch）
- `IR = Memory[PC]`
  把 `PC` 指向的指令读入 `IR`
- `PC = PC + 4`
  `PC` 指向下一条指令（32位MIPS中一条指令占4字节）

#### 2: ID — 译码 / 读寄存器（Instruction decode/register fetch）
- `A = Reg[IR[25-21]]`
  读出 `rs` 寄存器的值到 `A`
- `B = Reg[IR[20-16]]`
  读出 `rt` 寄存器的值到 `B`
- `ALUOut = PC + (sign-extend(IR[15-0]) << 2)`
  提前算出分支目标地址（for `beq`）：
	#### 第一步：取出立即数：`IR[15-0]` 是指令里的 16 位立即数字段。
	第二步：符号扩展`sign-extend(IR[15-0])`  把 16 位扩展成 32 位，并保留正负号。 如果立即数是正数，前面补 0，如果是负数，前面补 1。对不涉及立即数的指令：因为此时还在译码，所以CPU不知道具体操作。反正ALU是空闲状态，万一这是一条分支指令呢？我先用闲置的 ALU 把分支地址算出来备用。反正不管是不是分支指令，电信号传过去算一下，也不多花时间。就算用不上，后续ALU计算直接覆盖掉就行了
	第三步：左移 2 位`<< 2` 相当于乘以 4。这是因为 MIPS 的分支偏移量是按 **字地址** 计算的，而内存是按字节编址，所以要乘 4 才能变成字节偏移。
	最后加上 `PC`，这样就得到分支目标地址：`目标地址 = PC + 偏移量`


#### 3: EX — 执行 / 地址计算 / 分支或跳转完成
- **R 型指令：**
  `ALUOut = A op B`
  比如加、减、与、或等

- **访存类指令：**
  `ALUOut = A + sign-extend(IR[15-0])`
  计算内存地址，<font color=orange>因为位数原因，这里的offset在符号位扩展后无需像beq在第二步一样*4</font>

- **分支指令：**
  `if (A == B) then PC = ALUOut`
  **如果两个寄存器相等**，就跳转

- **跳转指令：**
  `PC = PC[31-28] || (IR[25-0] << 2)`
  组成新的跳转地址

#### 4: MEM — 访存或 R 型完成
- **R 型指令：**
  `Reg[IR[15-11]] = ALUOut`
  把 ALU 结果写回目的寄存器

- **Lw 指令：**
  `MDR = Memory[ALUOut]`
  从内存读数据

- **Sw 指令：**
  `Memory[ALUOut] = B`
  把数据写入内存

#### 5: WB — 写回（Memory read completion）
- **Lw 指令：**
  `Reg[IR[20-16]] = MDR`
  把读出的数据写回寄存器

# X.4.有限状态机
### 1. 有限状态机的基本定义与角色

- **定义**：有限状态机由一组状态、一个根据当前状态和输入决定下一个状态的“次态函数”，以及一个决定控制信号值的“输出函数”组成。
- **角色**：在多周期处理器中，控制单元需要根据**指令类型**和**当前时钟周期**来生成不同的控制信号，FSM正是实现这一逻辑的关键手段。它将单周期中复杂的逻辑拆解到多个周期中按需执行。

### 2. 控制器的硬件实现
- **结构**：FSM控制器通常由**组合逻辑电路**（用于实现次态和输出函数）和一个**状态寄存器**（用于保存当前状态编号）组成。
- **输入输出**：
    - **输入**：来自指令寄存器（IR）的操作码（Opcode）和当前状态寄存器的值。
    - **输出**：用于控制数据通路的功能信号（如 `RegWrite`、`ALUSrcA` 等）以及指向下一状态的信号。

### 3. 指令执行的状态转换逻辑

- **公共起始阶段**：所有指令的前两个周期相同，即状态0（取指）和状态1（指令译码/读寄存器）。
- **分支路径**：根据操作码，FSM会进入不同的分支：
    - **访存指令（lw/sw）**：经历内存地址计算（状态2）、内存访问（状态3或5）以及写回寄存器（仅lw，状态4）。
    - **R型指令**：执行运算（状态6）后进行结果写回（状态7）。
    - **分支指令（beq）**：完成分支条件判断和PC更新（状态8）。
    - **跳转指令（j）**：直接更新PC地址（状态9）。

### 4. 控制信号的确定
- **真值表**：文档提供了16路数据通路控制输出与10个状态（0-9）之间的对应关系表。
- **ALU控制**：FSM通过 `ALUOp` 信号指导ALU在不同周期执行特定操作。例如，在第一和第二周期（取指和地址计算）通常执行加法（00），而在执行 `beq` 指令的第三周期则执行减法（01）。

### 5. 故障分析与练习
- PDF中还包含了针对FSM故障的思考题。例如，如果FSM的 `AluOp` 信号恒输出为00，或者 `RegDst` 信号恒为1，会分析哪些指令（如R型指令或lw）将无法正常工作，以此加深对FSM控制逻辑的理解。
