## 1.1 RISC vs CISC

**CISC**（Complex Instruction Set Computer，复杂指令集计算机）
- **指令多，功能强**
- 程序的各条指令是按顺序串行执行的。
- 结构复杂、面积大、功耗大。
- 例子：**X86**

**RISC**（Reduced Instruction Set Computer，精简指令集计算机）
 - 指令少，功能弱
- 简单高效
- 面积小、功耗低
- **指令长度通常是固定的**
- 例子：**ARM**

总的来说，CISC的设计目标是通过硬件来减少软件的复杂性，而RISC则试图简化硬件以提高性能。目前，许多现代处理器，如Intel的Pentium系列，都采用了CISC和RISC的混合架构。这些处理器在接收到CISC指令后，会将其分解为RISC指令，以便在同一时间内执行多条指令。这种混合架构的目标是结合CISC和RISC的优点，从软件和硬件方面取长补短。

#### B.溢出判断
 - 正+正=负，负+负=正

#### B2.双符号位法

| 结果符号位 | 含义       |
| ----- | -------- |
| 00    | 正数，无溢出 ✓ |
| 11    | 负数，无溢出 ✓ |
| 01    | 正溢出 ✗    |
| 10    | 负溢出 ✗    |

示例1：正溢出
```
00 1111111 (+127)
00 0000001 (+1)
--------------
01 0000000 ← 符号位01，正溢出！
```
示例2：负溢出
```
11 0000000 (-128)
11 0000001 (-127)
--------------
10 0000001 ← 符号位10，负溢出！
```

## 1.4 浮点数
### 1.4.1 表示方式
<font color=red>期末必考：十进制和IEEE754单精度浮点数的相互转化计算</font>
#### A.IEEE-754 32-bit单精度浮点数表示基础
浮点数采用科学计数法的形式：$N = F×R^E$

|  符号   |  含义   | 说明         |
| :---: | :---: | :--------- |
| **F** |  尾数   | 表示底数，决定精度  |
| **R** |  基数   | 表示这个数的进制   |
| **E** | 阶码/指数 | 表示数量级,决定范围 |
比如0.2D=2×10^(-1)，0.0101B=1.01×2^(-2)
```
| 符号位(S) | 指数(E) | 尾数(F) |
```
- **符号位**：1位（0正1负），表示整个浮点数的符号
- **指数**：8位，E，无符号数
- **尾数**：23位，定点小数，通常规格化

#### B.规格化
当E=1~254时：
$$N=(-1)^S×1.F×2^{E-127}$$
规格化本质上是把这个浮点数N的尾数部分固定成`1.XX`的格式，小数点左边永远是1。因为这个固定格式，因此这个前导1不在23位尾数中，而是省略&默认存在，这也是表达式中`1.F`的来历
表达式中127是因为表示指数E的有8位二进制，除开有其他用途的0和255共有254个位置，但指数有正有负，砍一半就是127,E-127就能让指数表示域的中点刚好为0

#### C.其他特殊情况
<font color=red>以下不属于规格化</font>
当E=0时：
	F=0表示±0，具体是正是负看符号位S
	F≠0表示**次正规数（Subnormal）**,表示“非常接近 0 的非零小数”，避免其突变成0$$N=(−1)^s×(0.f)×2^{−126}$$
		在规格化的情况下，最小的精度是$1.0×10^{-126}$，涉及到比这还小的数无法表示。为了部分解决这个问题，在Subnormal中取消了前导1，使得N能够表示$0.X×10^{-126}$这个数量级的浮点数，以牺牲一位精度为代价扩大表示范围

当E=255时
	F=0：表示无穷大
	F≠0：表示`NaN`(Not a Number)
