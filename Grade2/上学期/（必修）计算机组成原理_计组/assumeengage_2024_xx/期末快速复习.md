可以，而且你这门课**非常适合“按题型突击”**。

我把你上传的 2020–2023 年 A/B 卷看了一遍，题型结构非常稳定：基本都是 **15 道选择题 30 分 + 4 道分析计算题 20 分 + 2 道综合题 20 分 + 2 道数据通路分析与设计题 30 分**。也就是说，**后面 50 分的大题几乎是固定套路**，抓住这些比通读教材更划算。

我先直接给你结论：

**想高分，不要按“章节”学，要按“题型模板”学。**  
你最该拿下的是这 6 类：

1. **CPI / CPU 执行时间**

2. **IEEE754 单精度浮点数**

3. **汉明码**

4. **MIPS 汇编大题**

5. **Cache 地址映射 / 命中率 / 最终状态**

6. **单周期、多周期数据通路控制信号题**

这几类在近几年卷子里反复出现。比如 2022 B 卷的 4 道分析题就是执行时间、IEEE754、汉明码、加法器进位表达式；2021 A/B、2020 A/B 也都反复考 CPI、浮点、汉明码、乘除法/乘法硬件流程。综合题又几乎固定是 **MIPS 翻译 + Cache**，最后的大题固定围绕 **单周期/多周期数据通路**。

### 第一优先级：数据通路分析与设计，30 分

**这部分是最值钱的。**

## 二、你只需要会做的“6 大题型模板”

## 1）CPI / 执行时间题：白给分

近几年一直有。题目形式一般是：

- 已知各类指令比例，求 CPI、执行时间

- 或已知指令数变成多少、CPI 变几倍，求新执行时间  

### 你只背两个公式

**总执行时间**  
[  
T = \frac{CPI \times IC}{f}  
]

**平均 CPI**  
[  
CPI = \sum (各类指令占比 \times 该类CPI)  
]

### 做题步骤

1. 先写平均 CPI

2. 再写执行时间公式

3. 单位统一，频率是 Hz，周期是 (1/f)

### 你要死背的多周期 MIPS 各类 CPI

常见是：

- lw：5

- sw：4

- R-type：4

- beq：3

- j：3

这个从 2021、2020 的答案里都能对上，算出来分别是 4、4.05、4.15 这种。

### 考场技巧

这种题**先做**，因为 2 分钟能拿满。

---

## 2）IEEE754 单精度：必练

2020–2023 基本都在考。给一个十进制数，比如 -0.875、-2.5、8.25、62.5，让你写单精度浮点格式。

### 固定步骤

1. **十进制转二进制**

2. **规格化**  
   [  
   \pm 1.xxxx \times 2^e  
   ]

3. **符号位 S**
   
   - 正数 0
   
   - 负数 1

4. **阶码 E = e + 127**

5. **尾数 M = 去掉前面的 1 后的小数部分**

6. 补满 23 位

### 你只要记住

单精度结构：

- 1 位符号

- 8 位阶码

- 23 位尾数

### 高频失分点

- 忘记“**规格化后前导 1 不存**”

- 阶码不是直接写 e，而是 **e+127**

- 小数转二进制不会

### 考场策略

不会全写也别空：

- 写出二进制

- 写出规格化

- 写出符号位和阶码  
  按评分标准，这样就能拿过程分。答案文件里每一步基本都单独给分。

---

## 3）汉明码：纯模板题

也是每年高频。给 4 位数据，要求写海明码，偶校验或奇校验。

### 你直接背布局

4 位数据 (D_0D_1D_2D_3) 时，7 位海明码常用位置：  
[  
P1\ P2\ D0\ P3\ D1\ D2\ D3  
]

### 分组

- P1 管：1、3、5、7 位

- P2 管：2、3、6、7 位

- P3 管：4、5、6、7 位

### 做题步骤

1. 先把数据位塞进去

2. 根据“偶校验/奇校验”分别求 P1 P2 P3

3. 最后按顺序写成 7 位码

### 你现在不用懂原理，只要会套

比如答案里反复就是这样分组算的。

---

## 4）MIPS 汇编大题：拿分关键

近几年综合题非常常见，就是把 C 代码翻成 MIPS。

### 你只需要会 5 种句型

#### A. 赋值

- `x = y + z` → `add`

- `x = y + 常数` → `addi`

#### B. if

- `if (a==b)` → `beq`

- `if (a!=b)` → `bne`

- `<` 常用 `slti` / `slt`

#### C. while

先判断，再跳转回去。

#### D. 函数调用

背这个壳子：

- 参数放 `$a0-$a3`

- 返回值在 `$v0`

- `jal func`

- 返回用 `jr $ra`

#### E. 栈保存

只要题里有**嵌套调用**，大概率要保存：

- `$ra`

- 用到的 `$s` 寄存器

### 必背模板：函数调用

```asm
addi $sp, $sp, -12
sw   $ra, 8($sp)
sw   $s0, 4($sp)
sw   $s1, 0($sp)

...  # 组织参数
jal func

lw   $s1, 0($sp)
lw   $s0, 4($sp)
lw   $ra, 8($sp)
addi $sp, $sp, 12
jr   $ra
```

你上传的答案里，2021、2022 的函数翻译几乎都在考这个套路。

### 临场判断规则

- **只调用一次函数**：未必需要保存很多东西

- **嵌套调用**：几乎必存 `$ra`

- **后面还要继续用某寄存器**：存 `$s0/$s1`

### 不会写时怎么混过程分

就写：

1. 开栈

2. 存 `$ra`

3. 参数搬运到 `$a0/$a1`

4. `jal`

5. 恢复栈  
   这种题过程分很多。

## 5）Cache 题：最值得硬背

这个几乎年年有，而且做法固定。

### 常见问法

1. Cache 有多少块

2. Index 位多少

3. Tag 位多少

4. 访问序列后最终状态

5. 命中率多少

6. 某主存块映射到哪个 Cache 块/组

### 统一公式

#### 块数

[  
Cache块数 = \frac{Cache容量}{块大小}  
]

#### Index 位

[  
Index位 = \log_2(Cache块数)  
]  
直接映射这样用；组相联先算组数。

#### 字节偏移位

[  
偏移位 = \log_2(块大小(字节))  
]

#### Tag 位

[  
Tag = 地址总位数 - Index - 偏移位  
]

### 访问序列题做法

#### 直接映射

- `块号 = 地址 / 块大小`

- `Cache 行号 = 块号 mod Cache块数`

- `Tag = 块号前面的高位`

然后一条一条写表，后来的覆盖前面的。

答案文件里就是按这个做的。

### 命中率

[  
命中率 = \frac{命中次数}{总访问次数}  
]

### 高频坑

- 题目是**按字节编址**还是**按字编址**

- 块大小是“4 个字”时，要先换成字节

- 直接映射和组相联别混

- 129 号**单元**先变成所在块号，再找组号

## 6）数据通路题：30 分大头，必须背套路

这个看着最难，其实最机械。

近几年出法包括：

- 单周期里 `LW / SW / BEQ / SUB` 的执行步骤

- 某条指令的控制信号

- 单周期为什么要分指令存储器和数据存储器

- 多周期里 `BEQ / J / LW` 的 RTL 过程

- 某些控制信号卡死为 0 或 1，哪些指令会坏  

### 你不用自己发明，只要背“指令执行步骤”

#### 单周期 LW

1. 取指，PC+4

2. 读寄存器 rs

3. 符号扩展立即数

4. ALU 计算有效地址

5. 读数据存储器

6. 写回 rt

#### 单周期 SW

1. 取指，PC+4

2. 读 base 寄存器

3. 读 source 寄存器

4. 符号扩展立即数

5. ALU 算地址

6. 写内存  
   这个步骤在评分标准里写得很明确。

#### 单周期 R-type / sub

1. 取指，PC+4

2. 读两个源寄存器

3. ALU 运算

4. 写回 rd  

#### 单周期 beq

1. 取指，PC+4

2. 读两个寄存器

3. 符号扩展立即数

4. 比较是否相等

5. 若相等，PC 跳到分支目标  

### 控制信号题怎么做

你不需要死理解全部，只要先按指令类型判断：

#### LW

- `ALUSrc=1`

- `MemRead=1`

- `MemtoReg=1`

- `RegWrite=1`

- `MemWrite=0`

#### SW

- `ALUSrc=1`

- `MemWrite=1`

- `RegWrite=0`

#### R-type

- `RegDst=1`

- `RegWrite=1`

- `ALUSrc=0`

- `MemRead=0`

- `MemWrite=0`

#### BEQ

- `Branch=1`

- `RegWrite=0`

- `MemRead=0`

- `MemWrite=0`

#### J

- `Jump=1`

2022 的单周期控制信号题甚至直接让你根据机器码/指令填表，答案也给了具体信号值。

### 控制信号故障题怎么秒杀

这种题其实就是一句话逻辑：

- `RegWrite` 错：凡是**不该写寄存器**的指令会坏

- `MemWrite` 错：凡是**不该写内存**的指令会坏

- `Branch` 错：可能乱跳

- `RegDst` 错：R 型 / lw 目的寄存器选错

- `MemtoReg` 错：lw 写回值错

- `PCWrite/PCWriteCond` 错：跳转/分支/取指更新 PC 出问题  
  评分标准里就是这么给的。

### 关于“单周期为什么性能差”

直接背：

- 单周期的周期长度由**最慢那条指令**决定

- 不同指令本来耗时不同，但都被迫用同一个长周期，所以浪费时间，性能差  

### 关于“为什么单周期要分指令存储器和数据存储器”

直接背：

- 一个周期内，像 lw 这种既要取指又要访存

- 若只有一个单端口存储器，一个周期内没法同时访问两次

- 所以要分成独立的指令存储器和数据存储器

## 三、选择题只背这张“高频速记表”

这部分别全学，直接背高频判断。

### 1. 程序翻译链

- 高级语言 → 编译器

- 汇编语言/助记符 → 汇编器

- 目标文件 + 库 → 链接器  
  这些在 2020–2022 选择题里反复出现。

### 2. 存储器

- **Cache 用 SRAM**

- **主存常用 DRAM**

- **DRAM 需要刷新**  

### 3. 组合逻辑 vs 状态单元

- 组合：ALU、加法器、比较器、多路选择器

- 状态：寄存器、存储器  

### 4. MIPS 常用寄存器

- `$sp`：栈

- `$gp`：静态数据区

- `$ra`：返回地址

- `$a0-$a3`：参数

- `$v0`：返回值  

### 5. 存储层次

- 大部分**容量**在低层

- 大部分**成本/速度**在高层  

### 6. TLB

- **页表项的 Cache**  

### 7. 补码

- **补码加减法，符号位一起算**

- 做大小比较时先看是按有符号还是无符号  

### 8. 直接映射 / 全相联

- **速度最快**常考全相联

- **实现最简单**是直接映射

- **命中率最高**常考全相联  
  不同卷子会从不同角度问。

## 五、考场上怎么做，最容易高分

### 推荐做题顺序

1. 先做 **分析计算题**

2. 再做 **综合题里的 Cache**

3. 再做 **MIPS 翻译**

4. 再做 **数据通路里你最熟的那一问**

5. 最后做选择题里不确定的

### 原因

因为选择题最容易卡住，但大题很多是**步骤给分**。从评分标准看，过程明显给分，比如浮点、汉明、Cache、数据通路、MIPS 都是分步给。

### 你不会时也一定要写

- 浮点：写十进制转二进制、规格化

- 汉明：写分组

- Cache：写块数、偏移位、Index、Tag 公式

- 数据通路：写“取指、读寄存器、ALU、访存、写回”

- MIPS：先写开栈、存 `$ra`、`jal`

空着最亏。
