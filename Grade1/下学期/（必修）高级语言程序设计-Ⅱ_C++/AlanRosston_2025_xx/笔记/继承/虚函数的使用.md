## 虚函数的作用

虚函数的主要作用是实现**动态绑定（Dynamic Binding）**，即在运行时根据对象的实际类型来调用对应的成员函数。这与非虚函数的**静态绑定（Static Binding）** 不同，静态绑定是在编译时确定调用哪个函数

### 虚函数

使用虚函数可以方便子类，`virtual` 放在定义最前面，子类重写需要用 ` override ` 关键字明确重写虚函数：
```cpp
// 定义父类
class Base {
public:
 virtual void show() {
 cout << "Base show" << endl;
 }
};

// 定义子类
class Derived : public Base {
public:
 void show() override { // override是C++11引入的，用于明确表示重写虚函数
 cout << "Derived show" << endl;
 }
};

int main() {
 Base* ptr = new Derived(); // 指针类型是Base, 指向的对象却是Derived
 ptr->show(); // 输出 "Derived show"，动态绑定
 delete ptr;
 return 0;
}
```
这里特别的是指针类型是 `Base`, 指向的对象却是 `Derived`

### 纯虚函数

没有实现的虚函数声明为**纯虚函数**，这种类就是**抽象类**，不允许被实例化为对象，只能被继承使用
```cpp
class AbstractBase {
public:
 virtual void show() = 0; // 纯虚函数
};
```

### 虚析构函数

**虚析构函数**（Virtual Destructor）是通过在类的析构函数前加上 `virtual` 关键字定义的析构函数。它的主要作用是确保在使用**多态**时，通过基类指针删除派生类对象时，能够正确调用派生类的析构函数，从而避免资源泄漏。

### 为什么需要虚析构函数

有虚函数的类的析构函数最好声明为**虚析构函数**，这保证用基类指针删除派生类对象时可以调用派生类的析构函数
```cpp
class Base {
public:
 virtual ~Base() {
 cout << "Base destructor" << endl;
 }
};

class Derived : public Base {
public:
 ~Derived() {
 cout << "Derived destructor" << endl;
 }
};

int main() {
 Base* ptr = new Derived();
 delete ptr; // 输出 "Derived destructor" 和 "Base destructor"
 return 0;
}
```

## 虚函数举例
下面的代码实现了一个绘图系统，重点是一个函数能够自动根据参数中的类的类型来决定如何绘制
> 一个函数控制多种子类的行为
```cpp
class Shape {
public:
 virtual void draw() const = 0; // 纯虚函数
 virtual ~Shape() = default; // 虚析构函数
};

class Circle : public Shape {
public:
 void draw() const override {
 std::cout << "Drawing a Circle" << std::endl;
 }
};

class Rectangle : public Shape {
public:
 void draw() const override {
 std::cout << "Drawing a Rectangle" << std::endl;
 }
};

void drawShape(const Shape* shape) {
 shape->draw(); // 通过基类指针调用派生类的函数
}

int main() {
 Circle circle;
 Rectangle rectangle;

 drawShape(&circle); // 输出 "Drawing a Circle"
 drawShape(&rectangle); // 输出 "Drawing a Rectangle"
 return 0;
}
```
