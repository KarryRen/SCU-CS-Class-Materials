### **拷贝构造函数介绍**

1. **定义**：
 - 拷贝构造函数是一种特殊的构造函数，用于通过同类型的另一个对象来初始化一个对象。
 - 形式通常为：
 ```cpp
 ClassName(const ClassName& other);
 ```
 - 其中，参数是当前类的引用，通常使用 `const` 修饰以防止修改源对象。

2. **默认行为**：
 - 如果没有显式定义拷贝构造函数，编译器会自动生成一个默认的拷贝构造函数。
 - 默认拷贝构造函数执行**浅拷贝**，即逐成员复制（按值复制每个成员变量）。

3. **使用场景**：
 - 当需要创建一个对象并将其初始化为另一个对象时，拷贝构造函数会被调用。例如：
 ```cpp
 ClassName obj1; // 默认构造函数
 ClassName obj2 = obj1; // 拷贝构造函数
 ```
> 注意与=重载区分！

3. **默认拷贝构造函数的局限性**：
 - 对于动态分配的资源（如指针），默认的浅拷贝可能会导致多个对象共享同一块内存，进而引发问题（如双重释放）。

---

### **深拷贝技术**

1. **定义**：
 - 深拷贝是指在拷贝对象时，不仅复制成员变量的值，还复制其指向的资源，确保每个对象拥有独立的资源。

2. **实现深拷贝**：
 - 如果类中包含动态分配的资源（如指针），需要手动实现深拷贝。
 - 通常需要：
 - 在拷贝构造函数中分配新的内存，并复制资源的内容。
 - 在析构函数中释放资源，避免内存泄漏。

3. **示例**：
 ```cpp
 class MyClass {
 private:
 int* data;

 public:
 // 构造函数
 MyClass(int value) {
 data = new int(value);
 }

 // 拷贝构造函数（深拷贝）
 MyClass(const MyClass& other) {
 data = new int(*other.data); // 分配新内存并复制内容
 }

 // 打印数据
 void print() const {
 std::cout << "Data: " << *data << std::endl;
 }

 // 析构函数
 ~MyClass() {
 delete data; // 释放动态分配的内存
 }
 };

 int main() {
 MyClass obj1(10);
 MyClass obj2 = obj1; // 调用拷贝构造函数

 obj1.print();
 obj2.print();

 return 0;
 }
 ```

4. **注意事项**：
 - 深拷贝可以避免多个对象共享同一块内存，从而防止潜在的错误（如双重释放、意外修改等）。
 - 深拷贝需要额外的内存分配操作，可能会带来性能开销。

---

### **在继承下的拷贝构造函数**

1. **父类和子类的关系**：
 - 子类继承父类时，子类的拷贝构造函数需要负责初始化父类的部分。
 - 如果子类没有显式定义拷贝构造函数，编译器会自动生成一个默认的拷贝构造函数，并调用父类的拷贝构造函数。

2. **显式调用父类的拷贝构造函数**：
 - 在子类的拷贝构造函数中，通常需要显式调用父类的拷贝构造函数以正确初始化父类部分。
 - 形式如下：
 ```cpp
 SubClass(const SubClass& other) : BaseClass(other) { // 显式调用!
 // 子类特有成员的拷贝
 }
 ```

3. **示例**：
 ```cpp
 class Base {
 protected:
 int* baseData;

 public:
 Base(int value) {
 baseData = new int(value);
 }

 // 父类的拷贝构造函数
 Base(const Base& other) {
 baseData = new int(*other.baseData);
 }

 virtual ~Base() {
 delete baseData;
 }

 void printBase() const {
 std::cout << "Base Data: " << *baseData << std::endl;
 }
 };

 class Derived : public Base {
 private:
 int* derivedData;

 public:
 Derived(int baseValue, int derivedValue) : Base(baseValue) {
 derivedData = new int(derivedValue);
 }

 // 子类的拷贝构造函数
 Derived(const Derived& other) : Base(other) { // 显式调用父类的拷贝构造函数
 derivedData = new int(*other.derivedData); // 深拷贝子类成员
 }

 ~Derived() {
 delete derivedData;
 }

 void printDerived() const {
 printBase(); // 打印父类的部分
 std::cout << "Derived Data: " << *derivedData << std::endl;
 }
 };

 int main() {
 Derived obj1(10, 20);
 Derived obj2 = obj1; // 调用子类拷贝构造函数

 obj1.printDerived();
 obj2.printDerived();

 return 0;
 }
 ```

4. **输出结果**：
 ```
 Base Data: 10
 Derived Data: 20
 Base Data: 10
 Derived Data: 20
 ```

5. **注意事项**：
 - 子类的拷贝构造函数必须显式调用父类的拷贝构造函数，否则父类部分不会被正确初始化。
 - 如果父类或子类中有动态分配的资源，必须确保深拷贝以避免共享同一块内存。

6. **默认拷贝构造函数的不足**：
 - 如果子类依赖于父类的默认拷贝构造函数，可能会导致浅拷贝问题。
 - 手动定义拷贝构造函数可以确保父类和子类的资源都被正确拷贝。

---

### **总结**

1. **拷贝构造函数**：
 - 用于通过同类型的对象初始化新对象。
 - 默认拷贝构造函数执行浅拷贝，可能导致问题。

2. **深拷贝**：
 - 通过手动分配内存和复制内容实现深拷贝，确保对象独立管理资源。
 - 深拷贝避免了共享内存引发的问题，但增加了性能开销。

3. **继承下的拷贝构造函数**：
 - 子类的拷贝构造函数需要显式调用父类的拷贝构造函数以初始化父类部分。
 - 必须确保父类和子类的资源都被正确拷贝，通常需要实现深拷贝。
